Bus

Apa Itu Bus ?

  • Bus data adalah lintasan yang digunakan untuk memungkinkan perpindahan data antara modul dalam sebuah sistem komputer.
  • Bus data terdiri dari sejumlah saluran yang berfungsi sebagai jalur transfer data. Jumlah saluran dalam bus data berkaitan dengan panjang word yang ditransfer, seperti 8, 16, atau 32 saluran.

Bus memiliki beberapa jenis, di antaranya adalah bus ,address bus , dan control bus.

  • Bus Data bus digunakan untuk mentransfer data antara CPU dan memori atau perangkat I/O.
  • Address bus digunakan untuk mengirimkan alamat memori atau perangkat I/O yang ingin diakses.
  • Control bus digunakan untuk mengontrol operasi bus seperti pembacaan dan penulisan data.

Data bus, address bus, dan control bus adalah tiga jenis bus yang membentuk sistem bus pada arsitektur komputer. Ketiga bus ini memiliki fungsi dan karakteristik yang berbeda satu sama lain.

Jenis - Jenis Bus

  1. Data Bus Data bus adalah jenis bus yang digunakan untuk mentransfer data antara CPU dan memori atau perangkat I/O. Data bus ini juga digunakan untuk menghubungkan CPU dengan bus ekspansi. Lebar bus data menentukan jumlah bit data yang dapat ditransfer pada satu waktu, sehingga semakin lebar bus data maka semakin cepat pula transfer data yang dapat dilakukan.
  2. Address Bus Address bus adalah jenis bus yang digunakan untuk mengirimkan alamat memori atau perangkat I/O yang ingin diakses oleh CPU. Lebar bus alamat menentukan jumlah bit alamat yang dapat diakses oleh CPU, sehingga semakin lebar bus alamat maka semakin banyak memori atau perangkat I/O yang dapat diakses oleh CPU.
  3. Control Bus Control bus adalah jenis bus yang digunakan untuk mengontrol operasi bus seperti pembacaan dan penulisan data. Control bus terdiri dari beberapa sinyal kontrol seperti sinyal pembacaan tulisan, sinyal request, sinyal acknowledge, dan sinyal interrupt. Sinyal kontrol ini digunakan untuk mengatur operasi bus dan menentukan tindakan yang harus dilakukan oleh CPU atau perangkat I/O.

Ketiga jenis bus ini bekerja bersama-sama untuk memungkinkan CPU untuk mengakses memori dan perangkat I/O pada sistem komputer. Data bus digunakan untuk mentransfer data, address bus digunakan untuk mengakses alamat memori atau perangkat I/O, dan control bus digunakan untuk mengatur operasi bus. Semua bus ini sangat penting dalam arsitektur komputer karena tanpa bus yang tepat, sistem komputer tidak dapat berfungsi dengan baik.

Bagian Saluran Sistem Bus

  • Saluran data
  • Saluran alamat
  • Saluran control

Saluran Data

  • Saluran data adalah bagian dari bus data yang digunakan untuk mengirimkan data antar modul dalam sebuah sistem komputer.
  • Saluran data memiliki lebar yang sama dengan lebar bus data, yaitu diukur dalam satuan bit seperti 8, 16, atau 32 bit.
  • Jumlah saluran data dalam bus data sangat penting karena mempengaruhi kecepatan transfer data antar modul. Semakin banyak jumlah saluran data, semakin cepat transfer data dapat dilakukan.
  • Transfer data dalam saluran data dilakukan dengan menggunakan sinyal listrik yang berbeda-beda. Sinyal ini digunakan untuk merepresentasikan bit data, baik dalam bentuk 0 atau 1.
  • Pada bus data yang lebih lebar, setiap bit dalam data ditransfer melalui saluran data yang berbeda. Oleh karena itu, semakin lebar bus data, semakin banyak saluran data yang dibutuhkan.

Saluran Alamat

  • Saluran alamat merupakan bagian dari bus pada komputer yang bertugas untuk mentransfer alamat memori atau perangkat I/O.
  • Saluran alamat mengirimkan alamat ke memori atau perangkat I/O agar dapat diakses oleh CPU atau perangkat lainnya.
  • Setiap alamat dalam memori atau perangkat I/O memiliki nomor unik yang dikenal sebagai alamat fisik.
  • Saluran alamat pada komputer memiliki lebar yang ditentukan oleh jumlah bit yang digunakan untuk mentransfer alamat. Semakin banyak bit pada saluran alamat, semakin besar pula ruang alamat yang dapat diakses.
  • Jumlah saluran alamat pada komputer juga mempengaruhi kapasitas maksimal memori yang dapat diakses oleh sistem komputer.
  • Selain digunakan untuk mengakses memori, saluran alamat juga digunakan untuk mengakses perangkat I/O seperti keyboard, mouse, printer, dan lain-lain.
  • Saluran alamat bekerja secara bersamaan dengan saluran data dan saluran kontrol untuk membentuk sebuah bus yang digunakan untuk mentransfer data antara CPU, memori, dan perangkat I/O.

Saluran Kontrol

  • Saluran kontrol berfungsi sebagai pengatur aliran data dan sinyal kontrol antara CPU dan perangkat lain di dalam komputer.
  • Saluran kontrol juga bertanggung jawab untuk mengirimkan sinyal kontrol seperti sinyal clock, reset, dan sinyal interrupt kepada perangkat lain.
  • Saluran kontrol terdiri dari berbagai macam sinyal kontrol yang dihasilkan oleh CPU seperti sinyal pembacaan data, sinyal penulisan data, sinyal pembacaan alamat, dan sinyal penulisan alamat.
  • Dalam sebuah komputer, saluran kontrol juga memiliki peran dalam mengatur penggunaan saluran data dan saluran alamat, sehingga data dapat diproses dengan benar.
  • Kecepatan saluran kontrol sangat penting dalam menentukan kinerja sebuah sistem komputer. Semakin cepat saluran kontrol, semakin cepat pula data dan sinyal kontrol dapat diproses oleh CPU dan perangkat lain di dalam sistem komputer.

Element Perancangan bus

Elemen perancangan bus pada arsitektur komputer terdiri dari beberapa komponen, yaitu : bus data, bus alamat, control bus, dan timing.

  • Bus data adalah saluran yang digunakan untuk mentransfer data dari satu perangkat ke perangkat lain.
  • Bus alamat adalah saluran yang digunakan untuk mentransfer alamat memori atau alamat port I/O untuk diakses oleh CPU atau perangkat lainnya.
  • Control bus adalah saluran yang digunakan untuk mengontrol bus data, bus alamat, dan perangkat-perangkat dalam sistem komputer.
  • Timing adalah mekanisme yang digunakan untuk mengatur kecepatan transfer data antar perangkat pada bus.

Selain itu, elemen perancangan bus juga dapat meliputi parameter dasar perancangan bus seperti lebar bus, jenis transfer data, dan metode arbitrasi. Lebar bus adalah jumlah bit yang dapat ditransfer pada satu waktu, sedangkan jenis transfer data adalah jenis operasi yang dilakukan pada bus seperti read, write, read-modify-write, dan lain-lain. Metode arbitrasi digunakan untuk menentukan prioritas antara perangkat yang menggunakan bus.

Jenis - jenis Bus

  • Dedicated Bus
  • Multiplexed Bus

Dedicated Bus

Dedicated bus adalah jenis bus pada arsitektur komputer yang dirancang untuk tujuan tertentu atau spesifik. Bus ini hanya digunakan untuk satu fungsi atau tujuan saja. Contohnya adalah bus yang digunakan untuk mentransfer data dari hard disk ke CPU pada komputer desktop atau laptop.

Dedicated bus atau bus yang didedikasikan untuk suatu tujuan tertentu memiliki beberapa keuntungan, di antaranya:

  • Kecepatan transfer data yang lebih tinggi: Karena dedicated bus hanya digunakan untuk tujuan tertentu, maka kecepatan transfer data pada bus tersebut biasanya lebih tinggi dibandingkan dengan bus yang digunakan secara bersamaan oleh beberapa perangkat.
  • Penanganan beban lebih mudah: Dengan menggunakan dedicated bus, perangkat yang terhubung ke bus tersebut dapat dengan mudah ditentukan dan diatur prioritasnya dalam melakukan transfer data, sehingga beban kerja pada bus dapat lebih mudah ditangani.
  • Lebih aman dan handal: Dengan menggunakan dedicated bus, risiko terjadinya konflik antar perangkat dalam melakukan transfer data dapat dikurangi atau bahkan dihilangkan, sehingga membuat bus menjadi lebih aman dan handal.

Multiplexed Bus

Multiplexed bus adalah jenis bus pada arsitektur komputer yang digunakan untuk mentransfer data, alamat, dan sinyal kontrol melalui satu saluran fisik. Pada bus multiplexed, saluran fisik dapat digunakan secara bergantian oleh beberapa sinyal, sehingga memungkinkan beberapa sinyal untuk digunakan pada waktu yang sama.

  • Hemat penggunaan pin pada CPU: Dalam bus multiplexed, beberapa saluran (channel) dapat digunakan untuk mengirimkan alamat dan data secara bergantian. Hal ini memungkinkan penggunaan pin pada CPU lebih hemat, karena tidak seluruh saluran (channel) akan digunakan pada satu waktu.
  • Menambah kapasitas data transfer: Dalam bus multiplexed, saluran data dapat digunakan secara bergantian dengan saluran alamat.
  • Mudah dikembangkan: Bus multiplexed relatif mudah dikembangkan karena dapat dilakukan dengan cara menyederhanakan jalur data dan alamat.

Metode Arbitrasi

Metode tersentralisasi pada perancangan bus mengandalkan satu pengontrol bus sentral atau arbiter untuk mengatur penggunaan bus oleh modul. Pengontrol bus sentral atau arbiter bertanggung jawab untuk memilih modul mana yang akan memiliki akses ke bus pada suatu waktu tertentu. Pengontrol bus sentral ini dapat berupa modul I/O atau bagian fungsi CPU. Setiap modul akan mengajukan permintaan untuk menggunakan bus melalui pengontrol bus sentral, dan pengontrol bus sentral akan memutuskan modul mana yang memiliki prioritas untuk mengakses bus terlebih dahulu.

Metode Timing-Asinkron

  • Metode Timing-Asinkron adalah metode pengiriman data di mana pengiriman dilakukan secara bebas waktu atau tidak terikat oleh clock atau sinyal timing yang ditentukan oleh master.
  • Metode ini memungkinkan setiap perangkat untuk mengambil dan mengirim data pada waktu yang berbeda-beda, sehingga meminimalkan konflik dan kegagalan dalam transfer data.
  • Dalam metode ini, setiap perangkat dalam sistem memiliki sinyal kontrol khusus untuk memulai dan menghentikan pengiriman data. Ketika sinyal ini diaktifkan, perangkat dapat mengirim atau menerima data pada waktu yang diinginkan.
  • Metode Timing-Asinkron digunakan pada banyak perangkat elektronik, seperti keyboard, mouse, printer, dan sebagainya.
  • Keuntungan lain dari metode Timing-Asinkron adalah bahwa ia memungkinkan pengiriman data secara fleksibel tanpa harus mempertimbangkan kecepatan clock yang berbeda pada masing-masing perangkat. Namun, metode ini juga memiliki kelemahan dalam hal kecepatan transfer data yang lebih lambat dibandingkan dengan metode sinkron.

Lebar Bus

  • Lebar bus menentukan berapa banyak bit yang dapat ditransfer dalam satu siklus clock. Semakin lebar bus, semakin banyak bit yang dapat ditransfer dan semakin cepat transfer data antar perangkat.
  • Lebar bus juga mempengaruhi performa sistem secara keseluruhan. Jika lebar bus terlalu kecil, transfer data antar perangkat akan menjadi lebih lambat dan dapat menghambat kinerja sistem.
  • Lebar bus biasanya diukur dalam bit dan dapat memiliki nilai yang berbeda, seperti 8 bit, 16 bit, 32 bit, atau bahkan lebih. Lebar bus yang lebih besar membutuhkan kabel yang lebih banyak dan dapat meningkatkan biaya produksi sistem.

Jenis - jenis transfer data

Operasi transfer data pada komputer merupakan proses penting yang memungkinkan pertukaran data antara berbagai modul komputer. Pada operasi baca (read), data diambil dari memori dan ditransfer ke CPU. Sebaliknya, pada operasi tulis (write), data ditulis ke memori dari CPU. Terdapat juga operasi kombinasi seperti operasi read-modify-write, dimana data yang diambil dari memori diubah pada CPU dan kemudian ditulis kembali ke memori. Agar semua operasi komputer dapat berjalan dengan baik, bus harus mampu menyediakan layanan saluran bagi semua operasi tersebut.

  • Peripheral Component Interconnect (PCI)
  • Universal Standard Bus (USB)
  • Small Computer System Interface (SCSI)
  • Bus P1394 / Fire Wire

Input/Output

Apa itu I/O?

Input & Output (I/O) adalah proses pertukaran data antara sistem komputer dan perangkat luar seperti keyboard, mouse, printer, hard drive, dan perangkat lainnya. I/O terjadi ketika data dipindahkan dari perangkat luar ke dalam sistem komputer atau sebaliknya, dari sistem komputer ke perangkat luar. Tujuannya adalah untuk memungkinkan sistem komputer berinteraksi dengan dunia luar, baik untuk menerima input dari pengguna maupun untuk mengirimkan output kepada pengguna atau perangkat lain. I/O adalah salah satu fungsi dasar dari sebuah sistem komputer dan menjadi bagian penting dalam perancangan arsitektur komputer.

Kategori:

  • Human Readable Machine : Perangkat fisik sebagai alat komunikasi manusia / pengguna komputer. (monitor, keyboard, mouse, printer, joystick, diskdrive)
  • Machine Readable Machine : Perangkat fisik yang berhubungan dengan peralatan / hanya bisa terbaca oleh komputer. (modul sensor dan tranduser untuk monitoring dan kontrol suatu peralatan atau system, Piringan disk).
  • Communication : Perangkat yang digunakan untuk komunikasi jarak jauh. (NIC, Modem).

I/O Port

I/O port adalah bagian dari modul I/O yang bertanggung jawab untuk menghubungkan perangkat eksternal dengan sistem komputer melalui jalur masukan (input) dan keluaran (output). I/O port terdiri dari beberapa saluran atau line yang dapat digunakan untuk mengirim atau menerima sinyal digital antara perangkat eksternal dan CPU. Setiap saluran pada I/O port memiliki fungsi yang berbeda-beda, misalnya saluran data untuk mentransfer data, saluran kontrol untuk mengatur operasi I/O, dan saluran status untuk memberikan informasi mengenai status perangkat eksternal. Dalam sistem komputer, I/O port penting karena memungkinkan perangkat eksternal seperti keyboard, mouse, printer, atau monitor dapat berkomunikasi dengan CPU dan berinteraksi dengan pengguna.

Konfigurasi I/O

Tempat penyimpan data sementara dalam CPU selama proses eksekusi. Apabila terjadi proses eksekusi data dalam register dikirim ke ALU untuk diproses, hasil eksekusi nantinya diletakkan ke register kembali.

AC / Unit control

AC/Unit Control adalah sebuah sistem kendali atau kontrol yang digunakan untuk mengatur dan mengendalikan suhu dan kelembaban di dalam ruangan dengan menggunakan AC (Air Conditioner) atau perangkat pendingin lainnya. Sistem kontrol ini biasanya terdiri dari beberapa komponen, seperti sensor suhu dan kelembaban, termostat, dan unit kontrol (control unit).

FGI & FGO Flags

FGI (Flag Global Interrupt) dan FGO (Flag Global Overflow) adalah dua flag yang terdapat pada mikrokontroler. FGI merupakan flag yang menandakan apakah ada permintaan interrupt yang sedang terjadi pada saat tertentu, sedangkan FGO merupakan flag yang menandakan apakah terjadi overflow pada hasil operasi aritmatika atau logika pada register tertentu. Kedua flag ini sangat penting dalam menjalankan program pada mikrokontroler, karena dapat mempengaruhi alur program dan operasi pada register.

Proses I/O (Input/Output)

Proses I/O (Input/Output) adalah proses pertukaran data antara CPU dan perangkat I/O. Berikut adalah beberapa poin yang terkait dengan proses I/O:

  • CPU akan mengirimkan perintah I/O melalui bus kontrol dan bus alamat ke modul I/O. Perintah ini memberi tahu modul I/O apa yang harus dilakukan, misalnya membaca data atau menuliskan data ke suatu perangkat.
    • Setelah menerima perintah, modul I/O akan mengatur pengiriman atau penerimaan data melalui port I/O yang terkait.
  • Data dikirim atau diterima melalui saluran data di bus data. Bus data menghubungkan modul I/O dengan CPU dan perangkat I/O.
  • Setelah selesai melakukan operasi I/O, modul I/O akan memberi tahu CPU bahwa operasi telah selesai melalui saluran kontrol di bus kontrol.
  • CPU dapat mengulangi proses I/O untuk perangkat yang sama atau untuk perangkat yang berbeda.

Proses I/O menjadi sangat penting dalam sistem komputer karena memungkinkan CPU untuk berkomunikasi dengan perangkat I/O seperti keyboard, mouse, printer, dan lain sebagainya.

Beberapa alasan peripheral tidak terhubung langsung dengan bus :

  • Kecepatan transfer data peripheral lebih lambat dari pada transfer data pada CPU maupun memori.
  • Format dan panjang data pada peripheral berbeda dengan CPU, sehingga perlu modul untuk menselaraskannya.
  • Peripheral seringkali beroperasi dalam lingkungan yang berbeda dengan CPU, seperti tingkat tegangan, kecepatan operasi, atau bahkan jenis komunikasi yang digunakan.
  • Peripheral seringkali memiliki sumber daya yang terbatas, seperti daya listrik atau memori
  • Metode operasi peripheral bervariasi ( tidak praktis apabila sistem komputer menangani berbagai macam sisem operasi peripheral ).
  • Adanya risiko interferensi atau gangguan elektromagnetik yang bisa terjadi apabila peripheral terhubung langsung dengan bus data dan alamat, sehingga perlu adanya isolasi atau penghalang yang mengurangi risiko tersebut.

Untuk melakukan operasi I/O, CPU akan mengirimkan alamat untuk mengakses modul I/O dan perangkat peripheral yang terhubung.

Perintah (Command) I/O

Operasi I/O dibagi menjadi empat klasifikasi, yaitu:

  • Kontrol: Melakukan aktivasi dan memberikan perintah kepada perangkat peripheral terkait.
  • Tes: CPU menguji kondisi, status, operasi, dan kesalahan modul I/O dan perangkat peripheral.
  • Baca: Mengambil data dari buffer internal untuk dikirim melalui bus setelah sinkronisasi kecepatan transfer data dilakukan.
  • Tulis: CPU memberikan perintah kepada modul I/O untuk mengambil data dari bus dan mengirimkannya ke perangkat peripheral yang dituju.

Teknik Operasi I/O

  1. Programmed I/O
  2. Interrupt-Driven I/O
  3. Direct Memory Access (DMA)

Programmed I/O

Programmed I/O adalah metode transfer data antara CPU dan peripheral di mana CPU secara langsung mengontrol transfer data. Dalam metode ini, CPU akan mengeluarkan perintah I/O untuk mengaktifkan perangkat peripheral dan mengambil atau menuliskan data ke dalam buffer internal. Proses ini dilakukan secara berurutan dan terus-menerus oleh CPU sampai transfer data selesai.

Kelemahan Programmed I/O terletak pada kinerja yang lambat karena adanya ketergantungan pada CPU untuk menangani seluruh proses pengiriman data. Hal ini dapat menghambat kinerja CPU dan memperpanjang waktu eksekusi program. Selain itu, penggunaan Programmed I/O juga membutuhkan banyak instruksi untuk mengatur transfer data, yang dapat menghasilkan overhead yang signifikan dan memperburuk kinerja sistem secara keseluruhan.

Proses :

  • CPU melakukan request operasi perintah kepada I/O
  • CPU mempunyai kontrol pada Sensing status, Read/write commands, dan Transfer data I/O
  • CPU memberi alamat, menentukan modul I/O dan perangkat peripheral
  • Modul I/O menjalankan operasi
  • Modul I/O memberikan status bits
  • Modul I/O tidak dapat melakukan interupsi kepada CPU
  • CPU memeriksa status bits secara periodik (I/O module memberikan informasi secara langsung ke CPU)

Interrupt-Driven I/O

  • CPU mengeluarkan perintah pada modul I/O, di saat yang sama CPU akan melakukan perintah lainnya. Setelah modul I/O selesai menjalankan instruksi, modul I/O akan melakukan interupsi pada CPU bahwa tugasnya telah selesai.
  • CPU melakukan multitasking beberapa perintah sekaligus, sehingga tidak ada waktu tunggu bagi CPU saat transfer data.Hal ini memungkinkan CPU untuk terus mengerjakan tugas-tugasnya tanpa harus menunggu operasi I/O selesai. Namun, penggunaan Programmed I/O juga memiliki beberapa kelemahan.
  • Salah satu kelemahan utamanya adalah overhead yang tinggi pada CPU karena CPU harus melakukan polling secara terus-menerus untuk memastikan apakah operasi I/O sudah selesai. Hal ini dapat mengakibatkan penggunaan CPU yang tidak efisien dan membatasi kinerja sistem secara keseluruhan.

Proses:

  • I/O controller menerima perintah, misal read.
  • I/O controller melaksanakan perintah pembacaan dari peripheral dan meletakkan paket data ke register data I/O controller, sementara CPU mengerjakan yg lain
  • I/O controller mengeluarkan sinyal interupsi ke CPU melalui saluran kontrol.
  • CPU meminta data.
  • Modul meletakkan data pada bus data dan modul siap menerima perintah selanjutnya.

Direct Memory Access (DMA)

Direct Memory Access (DMA) adalah metode transfer data antara perangkat peripheral dan memori komputer yang dilakukan tanpa melibatkan CPU. DMA memungkinkan perangkat peripheral untuk melakukan transfer data langsung ke atau dari memori tanpa melalui CPU, sehingga membebaskan CPU dari tugas transfer data dan memungkinkan CPU untuk melakukan tugas-tugas lainnya selama transfer data berlangsung.

Proses :

  • CPU mememberikan perintah ke modul DMA ( Read / write, Alamat perangkat I/O , Lokasi awal memori, jumlah word )
  • CPU melanjutkan pekerjaan lainnya
  • Modul DMA memindahkan seluruh blok data secara langsung ke memori tanpa harus melalui CPU.
  • Modul DMA mengirimkan signal interrupt ke CPU
  • CPU dapat menjalankan proses lainnya

DMA configuration

DMA configuration adalah pengaturan sistem DMA yang terdiri dari beberapa parameter yang harus diatur agar DMA dapat beroperasi dengan baik. Berikut adalah beberapa poin penting dalam DMA configuration:

  1. DMA Channel: Setiap DMA controller memiliki beberapa saluran DMA yang dapat digunakan untuk transfer data. Konfigurasi DMA harus menentukan saluran DMA yang akan digunakan untuk setiap transfer.
  2. Alamat sumber dan tujuan: DMA configuration juga harus menentukan alamat sumber dan tujuan untuk transfer data. Ini mengacu pada lokasi memori atau perangkat I/O tempat data akan ditulis atau dibaca selama transfer.
  3. Mode transfer: Ada dua mode transfer DMA, yaitu mode blok dan mode siklus. Mode blok adalah transfer DMA satu kali dari blok data yang besar, sedangkan mode siklus adalah transfer DMA berulang kali dari blok data yang lebih kecil.
  4. Ukuran transfer: Ukuran transfer DMA harus ditentukan agar DMA controller tahu berapa banyak data yang harus ditransfer. Ukuran transfer dapat diatur dalam byte, kata, atau double kata.
  5. Alamat sinkronisasi: DMA controller juga memerlukan alamat sinkronisasi yang akan diaktifkan ketika transfer selesai. Ini memungkinkan CPU untuk mengetahui kapan transfer DMA telah selesai.
  6. Prioritas DMA: Jika lebih dari satu saluran DMA digunakan, maka prioritas DMA harus ditentukan untuk menentukan saluran mana yang akan digunakan terlebih dahulu.
  7. Jenis transfer: DMA configuration juga harus menentukan jenis transfer yang akan dilakukan. Transfer dapat berupa baca tulis dari memori ke perangkat I/O atau sebaliknya.
  8. Mode interupsi: DMA configuration juga harus menentukan mode interupsi DMA. DMA dapat menghasilkan interupsi setelah transfer selesai atau setiap kali sejumlah data ditransfer.

Untuk menggunakan peripheral dibutuhkan 2 modul yaitu:

  1. Modul hardware (I/O Controller) :

    2. Interface (antarmuka) peripheral device ke sistem (CPU/memori).

  2. Modul Software

    3. Menyampaikan sejumlah perintah (command) ke I/O controller untuk melakukan operasi I/O.

Fungsi I/O Controller (I/O Module)

Interface Bus system yang dimaksud adalah bus controller atau biasa disebut sebagai bus bridge. Bus controller berperan sebagai central switch yang mengontrol dan mengatur komunikasi antara perangkat-perangkat peripheral (I/O device) dengan bus system, serta antara peripheral dengan memori utama atau register CPU. Bus controller memiliki saluran data, alamat, dan kontrol, yang digunakan untuk mentransfer informasi antara komponen-komponen tersebut

  • Control & timing :

    • Mengatur sistem keseluruhan dengan mensinkronkan kerja komponen komputer.

  • Data Buffering

    • Menyesuaikan perbedaan kecepatan transfer data antara perangkat peripheral dengan CPU.

  • Error Detection

    • Proses pada modul I/O dalam melaporkan kesalahan perangkat peripheral.

  • CPU Communication

    • Fungsi komunikasi antara CPU dan modul I/O dengan melibatkan control and timing arbitrasi bus.

  • Device Communication

    • Kontrol pemindahan data dari peripheral ke CPU melalui sebuah modul I/O.

Contoh operasi I/O driver:

  • mencetak baris melalui printer
  • membaca file dari floppy disk
  • menampilkan isi dari suatu lokasi memori
  • menyimpan isi memori ke hard disk
  • Membaca input dari keyboard
  • Mengirim data melalui jaringan (network)
  • Mengakses perangkat USB seperti flash drive atau mouse
